The General Subordination Ordering web application is an application of structural
modeling. This document contains development goals, objectives and notes. It is
intended to be a repository of design information that may be used to gain insight
into the web application architecture.

The primary difference between the strict subordination ordering web application
and the general subordination ordering web application is the fact that more than one
city may occupy the a given level in the general subordination ordering application.

This primary difference imposes some new functional requirements on the general
subordination ordering application. These new functional requirements are:

  • FR(1) - The ability to indicate that two or more cities are at the same level.
  • FR(2) - The ability to swap a group of cities that are all at the same level.
  • FR(3) - The ability to set a default level indicator.
  • FR(4) - The ability to select a default solution strategy.

Design details associated with these new functional requirements are outlined in
